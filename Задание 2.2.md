### Задание 2.2

| **№** | **Модель данных**       | **Преимущества**                                                                                                                                   | **Недостатки**                                                                                                                                                     |
| ----- | ----------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| 1     | **Иерархическая**       | Высокая скорость поиска данных при спуске по дереву от корня к листьям; простота структуры для иерархических данных.                               | Крайне жесткая структура; невозможность реализации связей "многие-ко-многим"; дублирование данных.                                                                 |
| 2     | **Сетевая**             | Способность эффективно моделировать сложные связи (многие-ко-многим) без избыточности; высокая скорость навигации по связям.                       | Очень высокая сложность проектирования и модификации базы данных; сложный язык запросов (навигационный).                                                           |
| 3     | **Реляционная**         | Универсальность (стандарт SQL); строгая математическая база; гарантия целостности данных (ACID); независимость данных от программ.                 | Относительная сложность горизонтального масштабирования; падение производительности при сложных `JOIN`-запросах на больших объемах данных.                         |
| 4     | **Документная (NoSQL)** | Гибкая схема (schema-less) — можно добавлять новые поля на лету; отличная горизонтальная масштабируемость; быстрый доступ к агрегированным данным. | Отсутствие стандартизированного языка запросов (каждая СУБД использует свой); более слабые гарантии консистентности (часто используется принцип BASE вместо ACID). |
