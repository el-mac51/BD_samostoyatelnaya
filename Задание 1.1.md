### Задание 1.1

| **№** | **Тип данных / объекты СУБД MySQL**   | **Описание / характеристики**                                                                                                                         |
| ----- | ------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------- |
| 1     | **INT (Тип данных)**                  | Целочисленный тип данных. Используется для хранения целых чисел (например, ID записей, количества). Может быть со знаком (signed) или без (unsigned). |
| 2     | **VARCHAR (Тип данных)**              | Строковый тип данных переменной длины. Оптимален для хранения текста, длина которого может меняться (имена, email, короткие описания).                |
| 3     | **DATETIME / TIMESTAMP (Тип данных)** | Типы для хранения даты и времени. TIMESTAMP часто используется для автоматической фиксации времени создания или изменения записи.                     |
| 4     | **Таблица (Объект)**                  | Базовый объект хранения данных в реляционной СУБД. Состоит из строк (записей) и столбцов (атрибутов), определяющих структуру данных.                  |
| 5     | **Индекс (Объект)**                   | Структура данных, повышающая скорость выполнения запросов (поиска) к таблице за счет дополнительных затрат памяти и времени на запись.                |
| 6     | **Представление / View (Объект)**     | Виртуальная таблица, содержимое которой определяется SQL-запросом. Упрощает сложные запросы и скрывает структуру базовых таблиц.                      |
| 7     | **Хранимая процедура (Объект)**       | Набор SQL-инструкций, сохраненный на сервере. Позволяет инкапсулировать бизнес-логику внутри БД и снизить сетевой трафик.                             |
